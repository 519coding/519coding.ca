What the Hell is a Syntax?

Whenever we speak or write, we always follow a certain set of rules that are required for a sentence to make sense. For example, "I like salami" makes sense, but "I salami" doesn't make much sense.

All programming languages also have a set of rules that need to be followed when writing code, and it's called syntax. Since computers have a hard time understanding semantics (The meaning behind text that isn't explicitly stated) and take everything literally, syntax becomes especially important. Thankfully, Python syntax is extremely simple in comparison to some other languages (like C++ ;0). However, there are still some special rules that need to be followed.

The Two Rules

Indents and Colons

The first major rule that needs to be followed is that blocks of code contained within function definitions, loops, conditional statements, and class definitions (Don't worry about these right now, we'll learn about them later) must follow a colon and must be indented.

Brackets

The second major rule is that all opening brackets must have its own distinct closing bracket.

Comments!

Comments are lines of text that aren't executed by the computer. They're useful because they can tell other programmers the reason why a piece of code was implemented (Remember: Others must be able to understand your code). Comments also allow you to temporarily remove a block of code without having to retype it, and also let you write reminders for yourself.

How's it Done?

You've actually already seen a comment! In Figure 1, the green text on line 1 is a comment. All single-line comments begin with a hash/pound/sharp sign (# will be called a hash symbol in these articles). Anything that comes after a hash symbol will be ignored.

Many other languages include multi-line comments, but Python doesn't technically have one. I say technically because you can use triple-quotation marks to convert a block of text into a string (You'll learn about strings soon) that will be ignored.

The Good, the Bad, the Ugly

Now, there are good comments, and there are not-so-good comments. Comments shouldn't tell you what the code does, that's usually clear to other programmers, comments should explain why a piece of code was written.

Print Functions

Unlike comments, print functions are read and executed by the computer. Print functions look like this: (You may have noticed them in Fig. 1, Fig. 2, Fig. 3, and Fig. 4)

As you can see, a print function is made of two parts: The word "print" on the left side and a pair of parantheses containing the message you wish to write on the right side. The word "print" tells the computer that you want to print a message and the parantheses let the computer know what it is you want to print. Speaking of printing, you might be asking yourself "Where does the computer print my message?" Well, you see, Python's interface is made up of two components: The editor and the compiler. The editor is where we type stuff and tell our computer what to do. The compiler is where the computer translates our code into a messsage. You can think of the editor as the place where we input code and the compiler as the place where the computer outputs its interpretation of whatever we inputted into the editor.
